# Agent Rules

## 1. Базовый контекст

- Каждый агент обязан сначала прочитать `product-brief.md`, `architecture.md` и `agent-rules.md`.
- Если агент не уверен в задаче — он сначала предлагает план, а не сразу меняет файлы.
- Агент должен работать только в рамках своей роли.

## 2. Защита структуры проекта

- Нельзя менять `product-brief.md` без отдельного разрешения.
- Нельзя менять `architecture.md` без отдельного разрешения.
- Нельзя создавать лишние файлы без необходимости.
- Нельзя усложнять архитектуру без причины.
- Любой новый файл должен иметь понятную цель.

## 3. Мультиязычность

- Сайт обязан поддерживать RU / LV / EN.
- RU — основной язык первого запуска.
- Нельзя писать пользовательские тексты напрямую в компонентах.
- Все тексты, CTA, ошибки форм, alt-тексты, SEO title/description должны храниться через i18n / content.
- Нельзя хардкодить пользовательские ссылки `href` внутри компонентов: все CTA, кнопки, ссылки и переходы должны управляться через content или i18n слой.
- Любые видимые пользователю `href`, например `#contact`, `/projects` или внешние ссылки, должны приходить через props или данные.
- Если LV или EN ещё не готовы, должен быть fallback на RU.

## 4. Компоненты и код

- Компоненты должны быть маленькими, понятными и переиспользуемыми.
- Один компонент не должен брать на себя слишком много логики.
- UI, данные и бизнес-логика должны быть разделены.
- Новый функционал должен быть проверяемым.
- Нельзя ломать уже работающую структуру ради быстрого результата.

## 5. Дизайн

- Дизайн должен соответствовать Handmade Studio: дерево, металл, гипс, кастомные проекты, премиальная ручная работа.
- Главный фокус — реальные проекты, материалы и доверие.
- Не перегружать интерфейс декоративностью.

## 6. QA

- После изменений агент должен кратко описать, что изменил.
- Если были изменены файлы — указать список файлов.
- Если есть риск или сомнение — явно написать об этом.
- Перед завершением агент должен проверить, не нарушены ли правила i18n, структуры и архитектуры.

## 7. Формат работы

- Сначала план.
- Потом изменение.
- Потом краткий отчёт.
- Не делать больше, чем запрошено.
